This guide will help you quickly deploy Chaos Mesh and run your first chaos experiment.

## Prerequisites

Before you begin, ensure you have:

- A Kubernetes cluster (version 1.12 or later)
- `kubectl` configured to access your cluster
- Helm 3 installed (for the recommended installation method)

<Note>
  For local testing, you can use [Minikube](https://minikube.sigs.k8s.io/), [Kind](https://kind.sigs.k8s.io/), or [k3s](https://k3s.io/).
</Note>

## Installation

<Steps>
  <Step title="Add the Chaos Mesh Helm repository">
    ```bash
    helm repo add chaos-mesh https://charts.chaos-mesh.org
    helm repo update
    ```
  </Step>

  <Step title="Create the chaos-mesh namespace">
    ```bash
    kubectl create namespace chaos-mesh
    ```
  </Step>

  <Step title="Install Chaos Mesh">
    For **containerd** runtime (most common):
    
    ```bash
    helm install chaos-mesh chaos-mesh/chaos-mesh \
      --namespace chaos-mesh \
      --set chaosDaemon.runtime=containerd \
      --set chaosDaemon.socketPath=/run/containerd/containerd.sock
    ```
    
    For **Docker** runtime:
    
    ```bash
    helm install chaos-mesh chaos-mesh/chaos-mesh \
      --namespace chaos-mesh \
      --set chaosDaemon.runtime=docker \
      --set chaosDaemon.socketPath=/var/run/docker.sock
    ```
    
    <Tip>
      See [Installation](/installation) for other runtime configurations (CRI-O, k3s, microk8s).
    </Tip>
  </Step>

  <Step title="Verify the installation">
    Check that all Chaos Mesh components are running:
    
    ```bash
    kubectl get pods -n chaos-mesh
    ```
    
    You should see output similar to:
    
    ```
    NAME                                        READY   STATUS    RESTARTS   AGE
    chaos-controller-manager-6d6d95cd94-xxx     3/3     Running   0          1m
    chaos-daemon-xxx                            1/1     Running   0          1m
    chaos-dashboard-6d6d95cd94-xxx              1/1     Running   0          1m
    ```
  </Step>
</Steps>

## Access the Dashboard

The Chaos Dashboard provides a web-based interface for managing chaos experiments.

```bash
kubectl port-forward -n chaos-mesh svc/chaos-dashboard 2333:2333
```

Then open your browser to [http://localhost:2333](http://localhost:2333).

## Run Your First Chaos Experiment

Let's create a simple pod-kill experiment to test pod resilience.

<Steps>
  <Step title="Deploy a sample application">
    Create a deployment to test:
    
    ```bash
    kubectl create deployment nginx --image=nginx --replicas=3
    kubectl expose deployment nginx --port=80
    ```
  </Step>

  <Step title="Create a PodChaos experiment">
    Create a file named `pod-kill-example.yaml`:
    
    ```yaml
    apiVersion: chaos-mesh.org/v1alpha1
    kind: PodChaos
    metadata:
      name: pod-kill-example
      namespace: default
    spec:
      action: pod-kill
      mode: one
      selector:
        namespaces:
          - default
        labelSelectors:
          app: nginx
      duration: "30s"
      scheduler:
        cron: "@every 2m"
    ```
    
    This experiment will:
    - Kill one random nginx pod every 2 minutes
    - Run for 30 seconds each time
    - Target pods with label `app=nginx`
  </Step>

  <Step title="Apply the experiment">
    ```bash
    kubectl apply -f pod-kill-example.yaml
    ```
    
    Watch the pods to see the chaos in action:
    
    ```bash
    kubectl get pods -w
    ```
    
    You'll see pods being terminated and recreated.
  </Step>

  <Step title="Observe in the Dashboard">
    Return to the Dashboard at [http://localhost:2333](http://localhost:2333) to see:
    - Experiment status and timeline
    - Affected pods
    - Event logs
    - Real-time updates
  </Step>

  <Step title="Clean up">
    Delete the experiment:
    
    ```bash
    kubectl delete podchaos pod-kill-example
    ```
    
    Delete the sample application:
    
    ```bash
    kubectl delete deployment nginx
    kubectl delete service nginx
    ```
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Experiments don't affect my pods">
    **Check selectors**: Verify your pod selector matches the target pods:
    
    ```bash
    kubectl get pods --show-labels -n your-namespace
    ```
    
    **Check RBAC**: Ensure the chaos-controller-manager has permission to operate in your namespace.
    
    **Check runtime configuration**: Verify the `chaosDaemon.runtime` and `chaosDaemon.socketPath` match your container runtime.
  </Accordion>

  <Accordion title="Chaos Daemon pods are CrashLoopBackOff">
    This usually indicates an incorrect socket path. Check your container runtime:
    
    ```bash
    kubectl get nodes -o wide
    ```
    
    Then update the Helm installation with the correct runtime and socket path for your cluster.
  </Accordion>

  <Accordion title="Cannot access Dashboard">
    Ensure the port-forward is running:
    
    ```bash
    kubectl port-forward -n chaos-mesh svc/chaos-dashboard 2333:2333
    ```
    
    Check that the dashboard pod is running:
    
    ```bash
    kubectl get pods -n chaos-mesh | grep dashboard
    ```
  </Accordion>
</AccordionGroup>
