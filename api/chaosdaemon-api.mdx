## Overview

This page documents the detailed message types used by the ChaosDaemon gRPC service. For service method documentation, see [gRPC API Overview](/api/grpc-overview).

## Network Chaos Types

### TcsRequest

<ParamField path="tcs" type="[]Tc">
  List of traffic control configurations to apply.
</ParamField>

<ParamField path="container_id" type="string">
  Target container ID.
</ParamField>

<ParamField path="enterNS" type="bool">
  Whether to enter container network namespace.
</ParamField>

### Tc

<ParamField path="type" type="Type">
  Traffic control type: `NETEM` or `BANDWIDTH`.
</ParamField>

<ParamField path="netem" type="Netem">
  Network emulation configuration (when type is NETEM).
</ParamField>

<ParamField path="tbf" type="Tbf">
  Token bucket filter configuration (when type is BANDWIDTH).
</ParamField>

<ParamField path="ipset" type="string">
  IPSet name for filtering.
</ParamField>

<ParamField path="protocol" type="string">
  Protocol to filter (tcp, udp, icmp, etc.).
</ParamField>

<ParamField path="source_port" type="string">
  Source port or port range.
</ParamField>

<ParamField path="egress_port" type="string">
  Egress/destination port or port range.
</ParamField>

<ParamField path="device" type="string">
  Network device name.
</ParamField>

### Netem

Network emulation parameters for delay, loss, corruption, etc.

<ParamField path="time" type="string">
  Base delay time (e.g., "100ms").
</ParamField>

<ParamField path="jitter" type="string">
  Delay variation/jitter (e.g., "10ms").
</ParamField>

<ParamField path="delay_corr" type="float">
  Delay correlation percentage (0-100).
</ParamField>

<ParamField path="limit" type="uint32">
  Queue limit in packets.
</ParamField>

<ParamField path="loss" type="float">
  Packet loss percentage (0-100).
</ParamField>

<ParamField path="loss_corr" type="float">
  Loss correlation percentage (0-100).
</ParamField>

<ParamField path="gap" type="uint32">
  Gap for loss pattern.
</ParamField>

<ParamField path="duplicate" type="float">
  Packet duplication percentage (0-100).
</ParamField>

<ParamField path="duplicate_corr" type="float">
  Duplication correlation percentage (0-100).
</ParamField>

<ParamField path="reorder" type="float">
  Packet reordering percentage (0-100).
</ParamField>

<ParamField path="reorder_corr" type="float">
  Reordering correlation percentage (0-100).
</ParamField>

<ParamField path="corrupt" type="float">
  Packet corruption percentage (0-100).
</ParamField>

<ParamField path="corrupt_corr" type="float">
  Corruption correlation percentage (0-100).
</ParamField>

<ParamField path="parent" type="TcHandle">
  Parent qdisc handle.
</ParamField>

<ParamField path="handle" type="TcHandle">
  This qdisc handle.
</ParamField>

<ParamField path="rate" type="string">
  Rate limit (e.g., "1mbit").
</ParamField>

### Tbf

Token Bucket Filter for bandwidth limiting.

<ParamField path="rate" type="string">
  Rate limit (e.g., "1mbit", "100kbit").
</ParamField>

<ParamField path="limit" type="uint32">
  Queue size limit in bytes.
</ParamField>

<ParamField path="buffer" type="uint32">
  Token bucket buffer size.
</ParamField>

<ParamField path="peak_rate" type="uint64">
  Peak rate for burst traffic.
</ParamField>

<ParamField path="min_burst" type="uint32">
  Minimum burst size.
</ParamField>

### TcHandle

<ParamField path="major" type="uint32">
  Major number of tc handle.
</ParamField>

<ParamField path="minor" type="uint32">
  Minor number of tc handle.
</ParamField>

### IPSetsRequest

<ParamField path="ipsets" type="[]IPSet">
  List of IPSet configurations.
</ParamField>

<ParamField path="container_id" type="string">
  Target container ID.
</ParamField>

<ParamField path="enterNS" type="bool">
  Whether to enter container network namespace.
</ParamField>

### IPSet

<ParamField path="name" type="string">
  IPSet name.
</ParamField>

<ParamField path="cidrs" type="[]string">
  List of CIDR ranges.
</ParamField>

<ParamField path="cidr_and_ports" type="[]CidrAndPort">
  List of CIDR and port combinations.
</ParamField>

<ParamField path="set_names" type="[]string">
  List of IPSet names for set-of-sets.
</ParamField>

<ParamField path="type" type="string">
  IPSet type (hash:ip, hash:net, hash:ip,port, etc.).
</ParamField>

### CidrAndPort

<ParamField path="cidr" type="string">
  CIDR range.
</ParamField>

<ParamField path="port" type="uint32">
  Port number.
</ParamField>

### IptablesChainsRequest

<ParamField path="chains" type="[]Chain">
  List of iptables chains to configure.
</ParamField>

<ParamField path="container_id" type="string">
  Target container ID.
</ParamField>

<ParamField path="enterNS" type="bool">
  Whether to enter container network namespace.
</ParamField>

### Chain

<ParamField path="name" type="string">
  Chain name.
</ParamField>

<ParamField path="direction" type="Direction">
  Chain direction: `INPUT` or `OUTPUT`.
</ParamField>

<ParamField path="ipsets" type="[]string">
  IPSet names to match.
</ParamField>

<ParamField path="target" type="string">
  Target action (DROP, ACCEPT, REJECT, etc.).
</ParamField>

<ParamField path="protocol" type="string">
  Protocol to match (tcp, udp, icmp, all).
</ParamField>

<ParamField path="source_ports" type="string">
  Source port or port range to match.
</ParamField>

<ParamField path="destination_ports" type="string">
  Destination port or port range to match.
</ParamField>

<ParamField path="tcp_flags" type="string">
  TCP flags to match.
</ParamField>

<ParamField path="device" type="string">
  Network device to match.
</ParamField>
